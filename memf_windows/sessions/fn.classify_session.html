pub fn classify_session(logon_type: u32, username: &str) -> bool
Expand description

Classify a logon session as suspicious based on forensic heuristics.

Returns true when any of the following conditions hold:

  • NetworkCleartext (type 8) logon — credentials sent in plaintext, common in legacy/misconfigured environments or credential harvesting
  • NewCredentials (type 9) with a non-system account — often indicates runas /netonly or pass-the-hash lateral movement
  • RemoteInteractive (type 10) from the SYSTEM account — SYSTEM should never initiate RDP sessions; indicates possible exploitation
