Module direct_syscalls

Expand description

Direct/indirect system call detection for EDR bypass analysis.

Detects processes using direct or indirect system call invocations to bypass EDR API hooks. When malware calls Nt* functions directly via the syscall/sysenter instruction instead of through ntdll.dll, it bypasses usermode hooks placed by security products.

Key techniques detected:

  • Direct syscall: The syscall instruction lives in non-ntdll code (SysWhispers, HellsGate, Halo’s Gate).
  • Indirect syscall: Code jumps into ntdll’s syscall gadget from a non-system module to make the return address appear legitimate.
  • Heaven’s Gate: 32-bit process transitions to 64-bit mode to invoke 64-bit NT syscalls directly, bypassing WoW64 layer hooks.
