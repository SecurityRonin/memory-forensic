Module cachedump 

Source
Expand description

Domain Cached Credential (DCC/MSCachev2) extraction from Windows memory dumps.

When domain users log in to a Windows machine, their credential hashes are cached in HKLM\SECURITY\Cache as NL$1, NL$2, … entries. These Domain Cached Credentials (DCC2/MSCachev2) can be extracted for offline cracking. This is the memory forensic equivalent of Volatility’s cachedump plugin.

The SECURITY hive cache is structured as: SECURITY\Cache\NL$1 — first cached credential entry SECURITY\Cache\NL$2 — second cached credential entry …up to NL$10 (typical maximum, configurable via CachedLogonsCount)

Each cache entry value contains a DCC2 header (96 bytes) followed by UTF-16LE encoded username and domain name strings.
