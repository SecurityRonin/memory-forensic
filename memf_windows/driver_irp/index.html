Module driver_irp

Expand description

Driver IRP dispatch table hook detection (Windows).

Each loaded Windows kernel driver has a _DRIVER_OBJECT containing a MajorFunction array of 28 function pointers — one per IRP major function code. Rootkits replace these pointers to intercept I/O operations (disk reads, network, filesystem). A hooked IRP handler points outside the driver’s own module address range.

Equivalent to Volatility’s driverirp plugin. MITRE ATT&CK T1014.
