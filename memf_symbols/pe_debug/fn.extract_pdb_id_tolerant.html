pub fn extract_pdb_id_tolerant(bytes: &[u8]) -> Result<PdbId>
Expand description

Extract PDB identification by scanning raw bytes for a CodeView RSDS record.

Tolerant fallback for extract_pdb_id: it does not parse the PE structure (section table, data directories, attribute-certificate table), so it works on an in-memory kernel image whose on-disk file offsets are no longer valid (goblin rejects such images with errors like “End of attribute certificates table is after the end of the PE binary”).

Scans bytes for the ASCII signature RSDS, then reads the 16-byte mixed-endian GUID, the 4-byte age, and the NUL-terminated PDB filename that follow. The first record with a non-empty, valid-UTF-8 filename wins.

All reads are bounds-checked; malformed or truncated input yields [Error::NotFound] rather than a panic (Paranoid Gatekeeper).
